Aufgabe 1

Wie wird ein Repository auf den Computer geklont?

Zuerst muss der Link von Repository kopiert werden. Danach wird dieser Link auf git Bash zusammen mit Funktion «git clone» eingetragen. Nach der Ausführung wird Repository lokal gespeichert.

Was bewirkt ein Commit?

Ein «commit» speichert die Veränderungen.

Was erzielt man durch einen Push?

Mit «push» können lokale commits auf remote-branch eingetragen werden.

Aufgabe 3

  • main: Der Zweck von "main" besteht darin, aktuellste sowie meist überprüfte Version zu behalten. Diese Version soll immer fehlerfrei sein.
  • develop: Dieser Branch wird zu Beginn eines Projekts erstellt und enthält Code mit neu entwickelten Features, die noch im getestet werden muss.
  • feature: "feature" Branch ist ein Beistandbranch und wird auf der Basis vom "develop"-Branch erstellt. Auf "feature" wird neue Features entwickelt und schlussendlich werden sie mit "develop" fusioniert bzw. gemergt.
  • release: Ein "release" Branch wird erstellt, wenn "develop" genug Features für ein neues Version hat. In diesem Branch werden nur Aufgaben, die kurz bevor Freigabe bewältigt werden sollen, wie Bugfixes, Dokumentationsgenerierung und Prüfungen aufgenommen werden.
  • hotfix: "hotfix" Branch wird erstellt, wenn es ein Problem auf "main" auftaucht. In diesem Branch wird nötige Bugfixes vorgenommen. "hotfix" ist für kleine Bugfixes nach dem Freigabe des aktuelles Versions zweckmässig.

Aufgabe 4

Git ist ein open-source Programm für Versionsverwaltung. Mit Git Repositorien und Git Workflow können Programmen zweckmässig bearbeitet und veröffentlicht werden. Ein Git Workflow besteht aus verschiedenen Repositorien. Sie werden auch "Branch" genannt. Mit Branchen können verschiedene Aufgaben unter Teams verteilt werden. Das ermöglicht ein organisiertes Arbeitsweise und vielseitiges Prozess.
GitHub ist ein netzbasierter Dienst für Git. Auf GitHub können Entwickler kooperativ an ihren Projekten arbeiten, sie speichern und veröffentlichen.
